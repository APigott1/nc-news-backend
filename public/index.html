/api/users

Responds with a json object containing all users.

Available Methods:

  • GET

Available Queries:

  • none

Example Response:


{ 
  "users": [
    {
      "username": "user_1",
      "name": "Real Name1",
      "avatar_url": "user1.jpg"
    },
    {
      "username": "user_2",
      "name": "Real Name2",
      "avatar_url": "user2.jpg"
    }
  ]
}

/api/articles

Responds with a json object containing all articles.

Available Methods:

  • GET

Available Queries:

Query Values Effect Default Value
sort_by any of the following: author, title, article_id, topic, created_at, votes, article_img_url, comment_count sorts the results based on the selected column, the default order is descending created_at
order either asc or desc decides which way the results are sorted desc
topic any topic found in the articles (coding, cooking, football) filters the results so that only topics of this query are returned N/A

Example Response:


{
  "articles": [
    {
      "author": "author1",
      "title": "Title Example1",
      "article_id": 34,
      "topic": "topic1",
      "created_at": "2020-11-22T11:13:00.000Z",
      "votes": 0,
      "article_img_url": "article1_img.jpg",
      "comment_count": 11
    },
    {
      "author": "author2",
      "title": "Title Example2",
      "article_id": 12,
      "topic": "topic2",
      "created_at": "2020-11-15T13:25:00.000Z",
      "votes": 0,
      "article_img_url": "article2_img.jpg",
      "comment_count": 7
    },
  ]
}

/api/articles/:article_id

Responds with a json object containing a specific article's information. The article_id parameter is a number representing the article_id of the article requested.

Available Methods:

  • GET
  • PATCH

Available Queries:

  • none

Request Body:

The PATCH method takes a body with a number of votes to attach to the article. Note: a negative number is allowed, in order to remove votes. 


{ "inc_votes": 100}

Example Response:

Both the GET and PATCH methods provide the following response (with the votes updated in PATCH's case). 


{
  "article": {
    "article_id": 1,
    "title": "Title Example",
    "topic": "topic1",
    "author": "author1",
    "body": "This is the body of the article...",
    "created_at": "2020-11-07T06:03:00.000Z",
    "votes": 0,
    "article_img_url": "article1_img.jpg",
    "comment_count": 8
  }
}

/api/articles/:article_id/comments

Responds with a json object containing a specific article's comments. The article_id parameter is a number representing the article_id of the article requested.

Available Methods:

  • GET
  • POST

Available Queries:

  • none

Request Body:

The POST method takes a body with a username and a body to attach to the article. The username is the author of the comment and the body represents the text of the comment. 


{ 
  "username": "comment_author",
  "body": "This is the comment text..."
}

Example Response:

Both the GET and POST methods will give the following response: 


{
  "comment": {
      "comment_id": 31,
      "article_id": 1,
      "body": "This is the comment text...",
      "votes": 11,
      "author": "comment_author",
      "created_at": "2020-09-26T16:16:00.000Z"
    }
}

/api/comments/:comment_id

Deletes a specific comment. The comment_id parameter is a number representing the comment_id of the comment to be deleted

Available Methods:

  • DELETE

Available Queries:

  • none

Example Response:

DELETE doesn't give a response body.
